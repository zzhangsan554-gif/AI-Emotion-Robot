## 一、项目概述

一个能在实验室/办公室巡逻的AI小车，能听、能看、能感知你的情绪，像一个有情感的AI伙伴。

核心卖点：

- 🎯 主动巡逻找到你（不是等唤醒）
- 🧠 大模型理解语义+语气（不是命令词）
- 👀 摄像头看表情+语音双模态情绪判断
- 🔧 自绘PCB硬件，完整软硬件系统
- 🗣️ 语音控制：说"退后""过来""停"直接执行
- 🤖 OpenClaw框架约束模型，安全可控

### 2.3 为什么这样设计？

**为什么Python和OpenClaw分开？**

- Python管硬件（串口/摄像头/麦克风），需要实时性
- OpenClaw管AI决策，需要约束模型防意外
- HTTP解耦，各管各的，不互相影响

**为什么用OpenClaw而不是裸调API？**

- 直接调GLM-4.7 API可能出现意外输出
- OpenClaw框架约束：通过skill+tools定义规则
- 有对话历史管理、abort打断机制
- 更安全可控

**为什么GLM-4-Voice在Python里而不是OpenClaw里？**

- GLM-4-Voice需要发送音频数据（base64编码）
- Python管麦克风和扬声器
- OpenClaw只处理文本

### 3.4 JY61P IMU

- 型号：维特智能JY61P（基于MPU6050，内置卡尔曼滤波）
- 输出：欧拉角（Roll/Pitch/Yaw），不用四元数
- 万向锁：本项目是地面平台，pitch几乎不到±90°，不受影响
- 通信：软件I2C，PB8(SCL) / PB9(SDA)（随便选GPIO模拟）
- 地址：0x50，读取频率：50Hz
- 作用：提供偏航角(朝向) + 加速度(碰撞检测) + 航位推算

### 3.5 超声波

- 3个HC-SR04，只看前方（左前/正前/右前）
- 自动巡逻时主要是前进，前方3个够用
- 后退和侧移是用户主动指令，用户自己判断安全

## 四、底层驱动（STM32端）

### 4.1 麦克拉姆轮逆运动学

STM32收到MOVE指令(speed, yaw)后，自己分解成4个轮速：

```c
void mecanum_inverse(float vx, float vy, float w,
                     float *v1, float *v2, float *v3, float *v4) {
    float L_plus_W = L + W;  // L=半轴距, W=半纵距
    *v1 = vx - vy - w * L_plus_W;  // 前左
    *v2 = vx + vy + w * L_plus_W;  // 前右
    *v3 = vx + vy - w * L_plus_W;  // 后左
    *v4 = vx - vy + w * L_plus_W;  // 后右
    // 归一化到 -1000~1000
}

// MOVE指令分解：
// vx = speed * cos(yaw_rad)   前进分量
// vy = speed * sin(yaw_rad)   侧移分量
// yaw=0 → 前进, yaw=90 → 右移, yaw=-90 → 左移, yaw=180 → 后退
```

### 4.2 编码器PID速度闭环

- 3个编码器用TIM2/TIM3/TIM4硬件读取（AB相正交解码）
- 第4个编码器用GPIO外部中断软件读取
- 每个电机独立PID控制，目标转速由逆运动学给出
- PID输出映射到PWM占空比(0~100%)

### 4.3 超声波避障

- 3个超声波轮流触发（避免串扰），每100ms一轮
- 前方障碍物<30cm → 自动减速
- 前方障碍物<10cm → 自动停止（即使有前进指令）

### 4.4 IMU数据读取

- 每20ms读取一次JY61P（50Hz）
- yaw角用于航向保持：巡逻时保持直线，转弯时精确控制角度
- pitch/roll用于检测异常倾斜（翻车保护）

## 五、通信协议

### 5.1 物理层

- USART3：PB10(TX) / PB11(RX)
- 波特率115200，8N1，无流控
- 3.3V电平（树莓派和STM32兼容，无需电平转换）

### 5.2 帧格式

```
┌──────┬──────┬──────┬──────────┬──────┬──────┐
│ 包头  │ 长度  │ 命令  │   数据   │ 校验  │ 包尾  │
│ 0xAA │ 1字节│ 1字节│  N字节   │ 1字节│ 0x55 │
└──────┴──────┴──────┴──────────┴──────┴──────┘

包头：0xAA
长度：命令码+数据+校验的字节数
命令码：1字节
数据：变长
校验和：长度+命令+数据 XOR
包尾：0x55
```

### 5.3 命令码

**下行（树莓派→STM32）：**

| 命令码  | 名称        | 数据                    | 说明                          |
| ---- | --------- | --------------------- | --------------------------- |
| 0x01 | MOVE      | speed(u16) + yaw(i16) | speed:0~1000, yaw:-180~180° |
| 0x02 | STOP      | 无                     | 紧急停止                        |
| 0x03 | SET_MODE  | mode(u8)              | 0手动/1巡逻/2跟随/3静默             |
| 0x05 | HEARTBEAT | 无                     | 心跳                          |

**上行（STM32→树莓派）：**

| 命令码  | 名称            | 数据                                                | 说明      |
| ---- | ------------- | ------------------------------------------------- | ------- |
| 0x81 | SENSOR_DATA   | dist_L(F)dist_F(F)dist_R(F)battery yaw pitch roll | 传感器10Hz |
| 0x83 | ACK           | cmd(u8)+result(u8)                                | 命令应答    |
| 0xFF | HEARTBEAT_RSP | 无                                                 | 心跳回复    |

通信频率：下行事件驱动，上行100ms(10Hz)，心跳1秒

## 六、AI语音层

### 6.1 完整流程

```
麦克风持续采集(16kHz/单声道/16bit)
    ↓
VAD检测（silero-vad，本地）
    → 检测到"说完了" → 截取音频片段
    ↓
声纹验证（3D-Speaker，本地，<200ms）
    → 不是主人 → 丢弃（省token！）
    → 是主人 → 继续
    ↓
GLM-4-Voice（云端API，zai-sdk，80元/百万tokens）
    → 输入：base64音频 + prompt
    → 同时返回：
      ① message.content（文本）
      ② message.audio['data']（音频）
    ↓
音频② → 扬声器播放（pygame，可被VAD打断）
文本① → 通过OpenClaw API发给OpenClaw
         → GLM-4.7判断是命令还是聊天
         → 命令 → exec curl调Python HTTP API → UART → STM32
         → 聊天 → 不执行
```

### 6.2 为什么声纹在GLM-4-Voice前面？

GLM-4-Voice每次调用80元/百万tokens。声纹验证在前面（本地免费）：

- 非主人说话 → 声纹失败 → 直接丢弃 → 省钱
- 主人说话 → 声纹通过 → 才调GLM-4-Voice

### 6.3 打断机制

- VAD持续监听（即使扬声器在播放）
- 用户说话 → VAD检测到 → 立即停止播放 → 新一轮流程
- OpenClaw收到新消息 → 自动abort上一轮请求
- 回声消除：初期"播放时降低VAD灵敏度"，后期可加WebRTC AEC

### 6.4 命令vs聊天区分

由OpenClaw的GLM-4.7通过function calling自动判断：

- "退后一点""快点过来""停" → 匹配move_robot/stop_robot → 执行
- "今天好累啊""你在干嘛" → 不匹配任何function → 纯聊天

> ⚠️ 注意：用户可能说"往前走""往前跑""快点快点再快一点"等各种表达
> GLM-4.7理解语义后自动映射到function参数，不需要穷举
> 以下命令映射表仅为示例：

```
用户说的话（示例）    GLM-4.7解析（示例）         说明
"往前走"             move_robot(speed=50, yaw=0)  正常前进
"快点快点"           move_robot(speed=80, yaw=0)  加速前进
"退后一点"           move_robot(speed=30, yaw=180) 慢速后退
"停"                 stop_robot()                 紧急停止
"跟我走"             set_mode("follow")           切换跟随
"去巡逻"             set_mode("patrol")           切换巡逻
"今天好累啊"         无function匹配               纯聊天
```

> ⚠️ 速度上下限待定：需要定义最低/默认/最高速度，后续实测后确定

### 6.5 音色规则（优先级）

```
1. 用户明确要求："用御姐音"/"换个声音" → 按用户要求
2. 用户没提音色 → 根据情绪自动选择：
   疲惫→温柔关心语速慢  开心→欢快活泼  愤怒→平和简短  悲伤→安慰关心
3. 默认 → 温柔御姐音
```

## 七、视觉层

### 7.1 视觉方案

**两个都部署，对比测试后选择：**

YOLO11-nano（主力）：

- 刀哥擅长（电赛国家二等奖）
- 人体检测精度高，12-18fps
- 可扩展检测其他物体

MediaPipe Pose（对比）：

- 轻量级，25-40fps（2倍帧率）
- 原生33个关键点，支持姿态识别
- 近距离效果好

部署策略：先部署YOLO11-nano，同时部署MediaPipe对比，根据体验选择

### 7.2 视觉进程工作流程

```
USB摄像头 → OpenCV采集(640x480, 30fps)
    ↓
YOLO/MediaPipe 人体检测
    ↓
提取人体位置(bounding box中心) + 估算距离(框大小)
    ↓
[可选] 表情识别 → happy/sad/angry/neutral
    ↓
写入消息队列(multiprocessing.Queue)：
  - person_pos: (x, y)
  - person_dist: 估算距离
  - expression: 表情
```

### 7.3 人体跟随

控制进程读取消息队列中的人体位置，PID控制移动：

```
人体偏左 → 小车左转    人体偏右 → 小车右转
人体太近 → 减速/停止   人体太远 → 加速靠近
人体消失 → 停止等待或切巡逻模式
```

### 7.4 情绪融合（视觉+语音双模态）

```
视觉情绪 == 语音情绪 → 置信度最高
视觉情绪 != 语音情绪 → 以语音为主（语气比表情更准）
只有视觉（没说话）   → 用视觉情绪
只有语音（看不到脸） → 用语音情绪
```

最终情绪 → 通知GLM-4-Voice调整回复语气和音色

### 7.5 姿态识别（优先级低，后期加）

MediaPipe关键点可识别：挥手打招呼/双手举高停下/指向某方向

## 八、系统整合

### 8.1 进程架构

```
Python主程序（单进程多线程）：
├── 主线程：Flask HTTP Server(:8080)，接收OpenClaw的指令
├── 语音线程：VAD → 声纹 → GLM-4-Voice → 发文本给OpenClaw
├── 视觉线程：摄像头 → 检测 → 写消息队列
├── 控制线程：读消息队列+传感器 → 决策 → UART → STM32
└── 传感器线程：读STM32回传的传感器数据(10Hz)
```

> 注：用多线程而非多进程，共享内存更简单（multiprocessing.Queue）

```
     │  "跟我走"           │ 检测到人
     ▼                     ▼
┌─────────┐           ┌─────────┐
│ 跟随模式 │           │ 交互模式 │
│(follow) │ ────────→ │(interact)│
└─────────┘  用户说话   └─────────┘
     │                     │
     │  "安静"             │ "安静"
     ▼                     ▼
┌─────────┐
│ 静默模式 │  只避障，不主动说话
│(silent) │
└─────────┘
```

- 巡逻：自主巡逻避障，检测到人主动打招呼
- 跟随：跟着前面的人走
- 交互：停下来跟人对话
- 静默：只执行避障，不主动说话

### 8.3 Python HTTP API（与OpenClaw对接）

```python
# Flask server :8080
POST /move   {"speed": 500, "yaw": 0}    → 发UART MOVE指令
POST /stop   {}                           → 发UART STOP指令
POST /mode   {"mode": 2}                  → 发UART SET_MODE指令
GET  /sensors                               → 返回最新传感器数据
```

OpenClaw的skill通过 exec curl 调用这些API

### 9.3 Skill定义（robot-control）

创建 `~/.openclaw/workspace/skills/robot-control/SKILL.md`：

```markdown
---
name: robot-control
description: 控制AI情绪感知巡逻陪伴机器人
---

# 机器人控制

当用户的指令涉及小车运动控制时，使用以下工具：

## 控制小车移动
用exec工具执行curl命令：
\`\`\`bash
curl -X POST http://localhost:8080/move -H "Content-Type: application/json" \
  -d '{"speed": 500, "yaw": 0}'
\`\`\`
speed: 0-1000（0=停止，1000=最快）
yaw: -180~180（0=前进，90=右移，-90=左移，180=后退）

## 紧急停止
\`\`\`bash
curl -X POST http://localhost:8080/stop
\`\`\`

## 切换工作模式
\`\`\`bash
curl -X POST http://localhost:8080/mode -H "Content-Type: application/json" \
  -d '{"mode": 1}'
\`\`\`
mode: 0=手动, 1=巡逻, 2=跟随, 3=静默

## 安全规则
- 速度默认不超过500（50%），除非用户明确要求加速
- 最高速度不超过800（80%）
- 任何不确定的指令都不要执行，先确认
```

### 9.4 Python和OpenClaw的对接方式

```
Python调GLM-4-Voice拿到文本
    ↓
Python通过OpenClaw的sessions_send API发送文本给OpenClaw
    ↓
OpenClaw(GLM-4.7)收到文本
    ↓
匹配function → skill告诉agent用exec curl调Python HTTP API
    ↓
Python HTTP API收到请求 → 发UART → STM32执行
```

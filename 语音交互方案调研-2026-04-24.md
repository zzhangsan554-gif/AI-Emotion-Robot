# 实时语音交互方案调研（2026-04-24）

> 刀哥的需求：找全双工+可挂API的语音模型，用于AI小车和精英班心伴

## 一、别人怎么做语音交互（真人感案例）

### 1. ClawStage（OpenClaw桌面精灵）⭐ 最相关
- **硬件**：树莓派5 + 旋转机构 + 摄像头 + 传感器
- **语音引擎**：声网（Agora）对话式AI引擎
- **核心体验**：
  - 真人级对话：毫秒级低延迟响应，捕捉语音细微情感
  - 端侧降噪 + 声纹识别（嘈杂环境也能听清）
  - 多模态情感连接：说话的同时转身体、转头看你
  - 80%丢包率下仍保持对话流畅
- **技术栈**：声网SD-RTN全球网络 + 嵌入式SDK + Thread/Matter协议
- **参考价值**：⭐⭐⭐⭐⭐ 跟我们的方案最接近！都是树莓派5 + 语音交互 + OpenClaw

### 2. 面壁智能 MiniCPM-o 4.5（开源全双工）⭐⭐⭐⭐
- 2026年2月4日发布
- 参数9B，**全双工多模态实时流机制**
- 无需回合制交互，同时看听说
- **开源！可以本地跑！**
- 端侧小模型，支持手机实时部署
- **参考价值**：⭐⭐⭐⭐ 如果树莓派5能跑9B模型，可以完全不依赖API！

### 3. EBO Air 2 Plus（家庭陪伴机器人）
- 赋之科技 + 声网合作
- 集成声网对话式AI引擎 + 双向视频通话
- 全球160+国家，80万用户
- **参考价值**：⭐⭐⭐ 商业化产品验证了声网方案的可行性

### 4. GLM-TTS（智谱开源TTS）⭐⭐⭐
- 流式推理：边生成边播放，延迟从8秒降到<1秒
- 支持声音克隆（3-10秒参考音频）
- KV Cache加速长文本生成
- 24kHz贪心采样：速度最快最稳定
- **参考价值**：⭐⭐⭐ 如果用GLM-4-Voice，TTS部分可以参考这个优化

### 5. 声网对话式AI引擎（通用方案）
- 设备端嵌入式SDK：轻量级、低功耗
- 端侧降噪 + 声纹识别
- 抗弱网80%丢包
- 5秒连通率99.5%
- **注意**：声网是RTC基础设施，不是语音大模型。需要自己接LLM
- **参考价值**：⭐⭐⭐ 如果延迟是首要考虑，声网是最佳传输层

## 二、技术方案对比

### 方案A：豆包端到端语音API（Seeduplex全双工）
- 全双工，边听边说
- 4种输入模式（麦克风实时/带静音/Push-to-Talk/纯文本）
- 支持System Prompt + 音色克隆
- 价格：~0.35元/分钟
- ⚠️ 需要企业认证（个人版只有半双工）

### 方案B：MiniMax Realtime API
- 全双工WebSocket
- 个人可用，注册即用
- 价格待确认
- ✅ 最快可以试起来的方案

### 方案C：MiniCPM-o 4.5 本地部署
- 9B参数，开源全双工
- 树莓派5可能跑不动（8GB显存不够）
- Windows电脑RTX4060可以跑
- ❌ 不适合移动小车，但可以当测试平台

### 方案D：声网 + LLM 级联
- 声网管传输和降噪
- 自己接GLM-4-Voice或其他LLM
- 延迟最优（声网全球网络）
- ⚠️ 声网需要付费，架构复杂

### 方案E：GLM-4-Voice（当前方案，半双工）
- 免费（Coding Plan）
- 需要自己处理VAD + 打断
- 半双工：说话时不能听
- ✅ 已验证可行

## 三、火山引擎企业认证流程

### 前置条件
1. 注册火山引擎账号（手机号即可）
2. 完成个人实名认证（身份证+人脸）

### 企业实名认证步骤
1. 登录控制台 → 账号中心 → 实名认证
2. 选择"企业实名认证"
3. 填写企业信息：
   - 企业名称（与营业执照一致）
   - 统一社会信用代码
   - 上传营业执照照片/扫描件
   - 法人姓名 + 身份证号
   - 法人身份证正反面照片
4. 法人手机号验证（短信验证码）
5. 可选：对公账户打款验证（火山引擎给你的公司账户打几分钱，你报回金额）
6. 提交审核 → **1-3个工作日**

### 所需材料清单
- ✅ 营业执照（拍照或扫描，清晰可读）
- ✅ 法人身份证正反面
- ✅ 法人手机号（能收验证码）
- ⚠️ 可能需要：对公账户信息（银行名称 + 账号）

### 认证后能做什么
- 开通豆包语音O2.0/SC2.0（Seeduplex全双工）
- 获取App ID + Access Key
- 100万token免费额度
- 按量付费

### 刀哥爸公司情况
- 注册资本10万
- ✅ 足够注册火山引擎企业认证
- 流程简单：主要是拍照上传营业执照 + 法人身份证

## 四、关键发现 & 建议

### ⭐ 最大发现：MiniCPM-o 4.5 开源全双工
- 9B参数，全双工多模态
- 如果能在Windows上跑，延迟和成本都最优
- 但树莓派5（8GB）大概率跑不动

### ⭐ 最佳路线：先MiniMax后豆包
1. **今天就能做**：注册MiniMax开放平台，试Realtime API
2. **并行推进**：让你爸公司注册火山引擎企业认证
3. **认证通过后**：切换到豆包Seeduplex（更成熟、免费额度更多）

### ⭐ 架构参考：ClawStage方案
- 跟你的AI小车架构最接近
- 声网SDK做传输层，大模型做决策层
- OpenClaw框架做Agent管理
- 树莓派5做硬件控制

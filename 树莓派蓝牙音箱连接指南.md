## 三、蓝牙音箱配对

### 3.1 命令行配对（推荐）

**步骤1：让音箱进入配对模式**
- 通常长按电源键3-5秒
- 指示灯快闪（表示配对模式）

**步骤2：扫描设备**
```bash
# 扫描蓝牙设备
sudo bluetoothctl scan on

# 查看扫描结果
sudo bluetoothctl devices

# 预期输出（找到你的音箱）：
# Device XX:XX:XX:XX:XX:XX:XX Bluetooth Speaker
```

**步骤3：配对设备**
```bash
# 进入bluetoothctl交互模式
sudo bluetoothctl

# 交互命令：
[bluetooth]# power on                    # 开启蓝牙
[bluetooth]# scan on                    # 扫描设备
[bluetooth]# pair XX:XX:XX:XX:XX:XX:XX    # 配对音箱（替换为你的音箱MAC地址）
[bluetooth]# connect XX:XX:XX:XX:XX:XX:XX   # 连接音箱
[bluetooth]# trust XX:XX:XX:XX:XX:XX:XX     # 设置为信任设备（下次自动连接）
[bluetooth]# exit                        # 退出
```

**验证连接：**
```bash
# 检查连接状态
sudo bluetoothctl info XX:XX:XX:XX:XX:XX:XX

# 预期输出：
# Connected: yes
```

## 五、Python蓝牙音箱播放

### 5.1 方案1：调用系统aplay（最简单）

```python
import subprocess
import os

def play_audio(audio_file):
    """
    播放音频文件到蓝牙音箱

    Args:
        audio_file: 音频文件路径（.wav或.mp3）
    """
    try:
        # 调用系统aplay播放
        subprocess.run(
            ['aplay', '-D', 'pulse', audio_file],
            check=True
        )
        print(f"播放成功: {audio_file}")
    except subprocess.CalledProcessError as e:
        print(f"播放失败: {e}")

# 使用示例
play_audio("/tmp/response.wav")
```

**优点：** 简单，不需要额外依赖

**缺点：** 无法精确控制音量

---

### 5.2 方案2：使用pygame（推荐 ⭐⭐⭐⭐）

```python
import pygame
import time

def init_bluetooth_speaker():
    """
    初始化pygame音频（使用PulseAudio默认输出）
    """
    pygame.mixer.init()

def play_audio(audio_file):
    """
    播放音频文件

    Args:
        audio_file: 音频文件路径
    """
    try:
        # 加载音频
        sound = pygame.mixer.Sound(audio_file)

        # 播放
        channel = sound.play()

        # 等待播放完成
        while channel.get_busy():
            time.sleep(0.1)

        print(f"播放完成: {audio_file}")
    except Exception as e:
        print(f"播放失败: {e}")

# 使用示例
if __name__ == "__main__":
    init_bluetooth_speaker()
    play_audio("/tmp/response.wav")
```

**优点：**
- 精确控制播放状态
- 支持暂停/停止
- 可以回调播放完成事件

### 5.3 方案3：使用PyAudio（最灵活）

```python
import pyaudio
import wave

def play_audio_wave(audio_file):
    """
    播放WAV文件（PyAudio）

    Args:
        audio_file: WAV文件路径
    """
    try:
        # 打开WAV文件
        wf = wave.open(audio_file, 'rb')

        # 初始化PyAudio
        p = pyaudio.PyAudio()

        # 打开音频流（连接PulseAudio）
        stream = p.open(
            format=p.get_format_from_width(wf.getsampwidth()),
            channels=wf.getnchannels(),
            rate=wf.getframerate(),
            output=True,
            output_device_index=get_bluetooth_device_index()  # 获取蓝牙设备索引
        )

        # 读取并播放
        data = wf.readframes(1024)
        while len(data) > 0:
            stream.write(data)
            data = wf.readframes(1024)

        # 清理
        stream.stop_stream()
        stream.close()
        p.terminate()

        print(f"播放完成: {audio_file}")
    except Exception as e:
        print(f"播放失败: {e}")

def get_bluetooth_device_index():
    """获取蓝牙音箱设备索引"""
    p = pyaudio.PyAudio()
    for i in range(p.get_device_count()):
        info = p.get_device_info_by_index(i)
        if "bluetooth" in info['name'].lower() or "bluez" in info['name'].lower():
            return i
    return None

# 使用示例
if __name__ == "__main__":
    play_audio_wave("/tmp/response.wav")
```

**安装PyAudio：**
```bash
pip install pyaudio
# 如果失败，先安装PortAudio
sudo apt install python3-pyaudio
```

**优点：**
- 最灵活，精确控制
- 支持任何音频格式

## 六、GLM-4-Voice音频播放集成

### 6.1 完整流程

```python
import pygame
import tempfile
import os
import base64

# 初始化pygame
pygame.mixer.init()

def play_glm4_voice_audio(audio_base64):
    """
    播放GLM-4-Voice返回的音频

    Args:
        audio_base64: GLM-4-Voice返回的base64编码音频
    """
    try:
        # 解码base64
        audio_data = base64.b64decode(audio_base64)

        # 保存到临时文件
        with tempfile.NamedTemporaryFile(suffix=".wav", delete=False) as f:
            temp_file = f.name
            f.write(audio_data)

        # 播放
        sound = pygame.mixer.Sound(temp_file)
        channel = sound.play()

        # 等待播放完成
        while channel.get_busy():
            time.sleep(0.1)

        # 删除临时文件
        os.unlink(temp_file)

        print("GLM-4-Voice音频播放完成")
    except Exception as e:
        print(f"播放失败: {e}")

# 使用示例（技术方案v3.0的集成）
audio_base64 = glm4voice_response['audio']['data']
play_glm4_voice_audio(audio_base64)
```

### 6.2 VAD配合（播放时降低灵敏度）

```python
def play_audio_with_vad(audio_file, vad):
    """
    播放音频，同时降低VAD灵敏度

    Args:
        audio_file: 音频文件
        vad: VAD对象
    """
    # 降低VAD灵敏度（避免回音误触发）
    vad.set_threshold(0.2)  # 播放时低灵敏度

    # 播放音频
    sound = pygame.mixer.Sound(audio_file)
    channel = sound.play()

    # 等待播放完成
    while channel.get_busy():
        time.sleep(0.1)

    # 恢复VAD正常灵敏度
    vad.set_threshold(0.5)  # 恢复正常

    print("播放完成，VAD恢复")
```

## 七、常见问题

### Q1：蓝牙连接不稳定怎么办？

**A：设置自动重连**
```bash
# 设置设备为信任（自动重连）
sudo bluetoothctl trust XX:XX:XX:XX:XX:XX:XX

# 开机自动连接
sudo bluetoothctl connect XX:XX:XX:XX:XX:XX:XX
```

---

### Q2：播放有延迟怎么办？

**A：调整音频缓冲区**
```python
# pygame设置小缓冲区
pygame.mixer.pre_init(44100, -16, 2, 512)  # 缓冲区512（默认2048）
pygame.mixer.init()
```

---

### Q3：音量太小怎么办？

**A：提高PulseAudio音量**
```bash
# 设置音量到100%
pactl set-sink-volume bluez_sink.XX_XX_XX_XX_XX 100%
```

---

### Q4：蓝牙音箱不工作怎么办？

**A：排查步骤**
```bash
# 1. 检查蓝牙服务状态
sudo systemctl status bluetooth

# 2. 检查连接状态
sudo bluetoothctl info XX:XX:XX:XX:XX:XX

# 3. 检查音频设备
pactl list sinks short

# 4. 查看PulseAudio日志
sudo journalctl -u pulseaudio -n 50
```

| 特性 | USB迷你音箱 | 蓝牙音箱 |

| 推荐度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

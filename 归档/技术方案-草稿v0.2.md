## 一、项目概述

### 做什么
一个能在实验室/办公室里巡逻的AI小车，能听、能看、能感知你的情绪，像一个有情感的AI伙伴。

### 核心卖点
- 不只是语音助手——它能**主动巡逻找到你**
- 不只是命令词识别——大模型**理解你的语义+语气**
- 不只是语音交互——摄像头**看到你的表情**，配合语音做双模态情绪判断
- 不只是软件——**自绘PCB硬件**，完整的软硬件系统

- **手脚** = STM32 + 电机（移动、转向）

## 三、数据流

### 场景A：你疲惫地说"我好累啊"

```
[1] 麦克风录到音频
    ↓ <100ms
[2] VAD检测到说完了
    ↓
[3] 声纹验证（3D-Speaker本地）→ 确认是主人 ✅
    ↓
[4] 视觉线程：表情识别→疲惫（同时进行，不等待）
    ↓
[5] Python拼接请求：
    音频（base64）+ prompt = "你是EmoBot，当前用户情绪疲惫，用温柔关心的御姐音回复，语速放慢"
    发给 GLM-4-Voice API
    ↓ 约1-3秒
[6] GLM-4-Voice返回音频 → 扬声器播放
    ↓ 同时
[7] 决策模块：情绪=疲惫 → 让小车缓慢靠近你
    发指令给STM32 → 电机慢速转动
```

### 场景B：巡逻时发现人

```
[1] 摄像头拍一帧（每秒15次）
    ↓ <50ms
[2] YOLO检测到人体
    ↓
[3] 计算人体相对位置和方向
    ↓
[4] 决策模块：切换到跟随模式
    ↓
[5] 发跟随指令给STM32 → 电机驱动小车跟人
    ↓（持续循环，不等语音模块）
```

### 场景C：语音控制

```
你说："过来"
    ↓ VAD + 声纹验证
    ↓ GLM-4-Voice理解意图
    ↓ 返回音频："好的，我过来了~"
    ↓ 同时：决策模块 → MOVE指令 → STM32 → 小车驶向你
```

**三个场景的线程独立运行，互不阻塞。**

## 四、软件模块详细设计

### 4.1 Python主程序（树莓派上运行）

#### 4.1.1 视觉模块
- **人体检测**：YOLO11-nano（刀哥擅长）
  - 输入：USB摄像头帧
  - 输出：人体边界框 + 中心坐标
  - 备选对比：MediaPipe Pose（更轻量）
  - 帧率目标：15fps
  
- **表情识别**：轻量CNN（FER2013预训练）
  - 输入：人脸区域
  - 输出：7类表情（开心/悲伤/愤怒/惊讶/恐惧/厌恶/中性）
  - 用现成预训练模型，不用自己训

- **人体跟随**：PID控制器
  - 画面中人体位置偏差 → 转向速度
  - 人体在画面中的大小 → 距离估算 → 前进后退

#### 4.1.2 语音模块

- **VAD**：silero-vad
  - 实时检测"有人说话"和"说完了"
  - 说完了才截取音频发给GLM-4-Voice

- **声纹验证**：3D-Speaker（阿里开源，本地运行）
  - 提前录入3-5句话注册主人声纹
  - 非主人声音直接忽略
  - 可随时添加/删除用户

- **GLM-4-Voice调用**：zai-sdk
  - 输入：base64编码的WAV音频 + 文本prompt
  - 输出：base64编码的WAV音频
  - 支持流式输出（stream=True，边生成边播放）
  - 通过prompt控制音色/情感/语速
  - 价格：80元/百万tokens（不在Coding Plan内）

- **音频播放**：pygame / simpleaudio
  - GLM-4-Voice返回的音频解码后直接播放

#### 4.1.3 决策模块

- **情绪→行为映射**：
  - 开心 → 小车转圈/靠近 + prompt"用欢快少女音"
  - 疲惫 → 小车缓慢靠近 + prompt"用温柔御姐音，语速放慢"
  - 烦躁 → 小车后退/安静 + prompt"用平和简短的语气"
  - 悲伤 → 小车靠近 + prompt"用安慰关心的语气"

- **状态机**：巡逻 → 跟随 → 交互 → 静默
  - 巡逻：自主巡航找人
  - 跟随：发现人后跟着走
  - 交互：人在说话时停下来回应
  - 静默：被叫"安静"后停止语音

#### 4.1.4 运动控制模块

- **UART协议**（树莓派→STM32）：
  - `MOVE:speed,angle\n`
  - `STOP\n`
  - `POLL:ultrasonic\n`
  
- **麦克拉姆轮运动学**：
  - 逆运动学：目标速度(vx, vy, ω) → 4个轮速
  - 4路PWM输出

### 4.2 STM32固件（C语言）

1. **电机控制**：4路PWM + 麦克拉姆轮逆运动学
2. **超声波**：3x HC-SR04（前方+左前+右前）
3. **UART通信**：接收树莓派指令，返回传感器数据
4. **电源管理**：ADC采集电池电压，低压告警

## 五、硬件设计

### 5.1 自绘PCB（嘉立创EDA）

**电机驱动板**：
- 主控：STM32F103C8T6
- 电机驱动：2x TB6612FNG（每个驱动2路电机）
- 电源：
  - 输入：12V锂电池
  - 12V→5V 降压（LM2596）→ 供电树莓派
  - 5V→3.3V LDO（AMS1117）→ 供电STM32
- 接口：
  - USB-TypeC（STM32烧录+调试）
  - 4x 电机接口（PH2.0端子）
  - 3x 超声波接口（排针）
  - UART排针（连接树莓派GPIO）
  - 电源开关 + 指示LED

### 5.2 机械结构

- **底盘**：3D打印或亚克力
- **麦克拉姆轮**：60mm x4
- **电机**：直流减速电机（TT马达或N20）
- **分层安装**：
  - 底层：电池+PCB+电机
  - 中层：树莓派
  - 顶层：摄像头+麦克风+扬声器

## 六、语音方案（已确定）

### ✅ GLM-4-Voice 端到端

```
麦克风音频 → GLM-4-Voice → 直接输出音频 → 扬声器播放
```

**核心优势：**
- 一个模型全干：ASR + 情绪理解 + 回复生成 + TTS 全内置
- 音色通过prompt控制："用温柔御姐音回复""用欢快少女音""语速放慢"等
- 不需要单独的ASR、TTS、情绪特征提取
- 支持流式输出（边生成边播放）
- 支持中英文、方言

**调用示例：**
```python
from zai import ZhipuAiClient
import base64, wave

client = ZhipuAiClient(api_key="your_api_key")

# 读取麦克风录制的音频
with open("input.wav", "rb") as f:
    audio_b64 = base64.b64encode(f.read()).decode()

response = client.chat.completions.create(
    model="glm-4-voice",
    messages=[{
        "role": "user",
        "content": [
            {"type": "text", "text": "你是EmoBot，用温柔御姐音回复，当前用户情绪疲惫"},
            {"type": "input_audio", "input_audio": {"data": audio_b64, "format": "wav"}}
        ]
    }],
    stream=True  # 流式输出
)

# 解码音频播放
for chunk in response:
    audio_data = base64.b64decode(chunk.choices[0].message.audio['data'])
    # 播放 audio_data...
```

**价格：** 80元/百万tokens（不在Coding Plan内，智谱有免费额度）

### 声纹验证

- **3D-Speaker**（阿里开源，本地运行）
- 提前录入3-5句话注册主人声纹
- 可随时添加/删除用户
- 非主人声音直接忽略

### 备选方案（如果GLM-4-Voice不够用）

- ASR：SenseVoiceSmall（硅基流动）
- LLM：GLM-4-Flash（免费模型，Coding Plan支持）
- TTS：CosyVoice2（硅基流动，支持音色克隆）

## 一、项目概述

一个能在实验室/办公室巡逻的AI小车，能听、能看、能感知你的情绪，像一个有情感的AI伙伴。

核心卖点：

- 主动巡逻找到你（不是等唤醒）
- 大模型理解语义+语气（不是命令词）
- 摄像头看表情+语音双模态情绪判断
- 自绘PCB硬件，完整软硬件系统
- 语音控制：说"退后""过来""停"直接执行

### 2.2 比喻理解

- **大脑(理解+说话)** = GLM-4-Voice（端到端语音模型，听懂+说话）
- **理智(决策+控制)** = OpenClaw + GLM-4.7（function calling，控制小车）
- **眼睛** = USB摄像头 + MediaPipe（看到人在哪、什么表情）
- **耳朵** = USB麦克风 + VAD + 声纹验证（听到主人说话）
- **手脚** = STM32 + 电机（移动、转向）
- **内耳(平衡感)** = JY61P IMU（知道朝向、姿态）

### 2.3 为什么需要两个模型？

- **GLM-4-Voice负责"听和说"**：端到端语音理解+生成，能理解语气/情绪，支持文本→音频(TTS)
- **OpenClaw(GLM-4.7)负责"思考和执行"**：支持function calling控制小车，Coding Plan免费

GLM-4-Voice不支持function calling → 需要OpenClaw来执行控制指令
OpenClaw不支持音频输入 → 需要GLM-4-Voice来理解语音

**结论：各司其职，搭配使用。**

## 三、语音交互完整流程

### 3.1 主流程

```
麦克风持续采集音频（16kHz, 单声道, 16bit）
         ↓
VAD检测（silero-vad，本地）
  - 检测到有人说话 → 开始录音
  - 检测到说完了 → 截取音频片段
         ↓
声纹验证（3D-Speaker，本地，<200ms）
  - 不是主人 → 直接丢弃（省GLM-4-Voice的token！）
  - 是主人 → 继续
         ↓
GLM-4-Voice（云端，1-3秒）
  - 输入：base64音频 + prompt
  - 同时返回：
    ① message.content（文本：理解了什么）
    ② message.audio['data']（音频：语音回复）
         ↓
音频② → 扬声器播放（pygame）
         ↓ 同时
文本① → OpenClaw（GLM-4.7）
  - 匹配function → function calling → UART → STM32控制小车
  - 不匹配function → 纯聊天，不执行任何动作
```

### 3.2 为什么声纹验证在GLM-4-Voice前面？

因为GLM-4-Voice每次调用都要花钱（80元/百万tokens）。

- 非主人说话 → 声纹验证失败（本地，免费）→ 直接丢弃 → 省钱！
- 主人说话 → 声纹验证通过 → 才调GLM-4-Voice → 花钱值得

### 3.3 打断机制

**场景：正在播放回复时用户突然说话**

```
[1] GLM-4-Voice返回音频 → 正在播放"好的，我过来了~"
[2] 用户突然说"停！"
[3] VAD一直监听 → 检测到新语音
[4] 立即停止当前音频播放
[5] 声纹验证 → GLM-4-Voice → 返回新文本"停"
[6] 文本给OpenClaw → OpenClaw自动abort上一轮 → stop_robot()
[7] 新音频"好的，停了" → 播放
```

**OpenClaw的abort机制：** 收到新消息时自动abort上一轮请求

**回声消除（防止把扬声器声音当用户语音）：**

- 初期方案：播放时降低VAD灵敏度（简单但粗糙）
- 后期方案：WebRTC AEC模块（专业方案，可后续加）

### 3.4 命令vs聊天区分

**GLM-4-Voice的prompt：**

```
你是EmoBot，一个AI陪伴机器人。
用户可能跟你日常聊天，也可能给你下命令。
不管用户说什么，你都用语音回复他。
但如果是命令，你回复的同时我会去执行。
回复要简短，1-3句话。
```

**OpenClaw的function定义（详见第七节）负责区分：**

- "退后一点" → 匹配move_robot → 执行控制
- "今天好累啊" → 不匹配任何function → 纯聊天

### 3.5 用户自定义音色 vs 情绪驱动的音色

**优先级规则：用户明确指令 > 情绪自动选择**

```
音色规则（优先级从高到低）：
1. 用户明确要求："用御姐音"/"换个声音" → 按用户要求执行
2. 用户没提音色 → 根据当前情绪自动选择：
   - 疲惫 → 温柔关心的语气，语速放慢
   - 开心 → 欢快活泼的语气
   - 愤怒 → 平和简短的语气，不要火上浇油
   - 悲伤 → 安慰关心的语气
   - 默认 → 温柔御姐音
```

## 六、UART通信协议

### 物理层

- 波特率：115200
- 数据位：8，停止位：1，校验：无，流控：无
- 电平：3.3V（树莓派和STM32都是3.3V，无需电平转换）

### 帧格式

```
┌──────┬──────┬──────┬──────────┬──────┬──────┐
│ 包头  │ 长度  │ 命令  │   数据   │ 校验  │ 包尾  │
│ 0xAA │ 1字节│ 1字节│  N字节   │ 1字节│ 0x55 │
└──────┴──────┴──────┴──────────┴──────┴──────┘

包头：0xAA（固定）
长度：从命令码到校验和的字节数（1字节）
命令码：1字节
数据：0~252字节（变长）
校验和：长度+命令+数据的所有字节XOR
包尾：0x55（固定）
```

### 命令码定义

**下行（树莓派→STM32，0x01~0x0F）：**

| 命令码  | 名称        | 数据格式                  | 说明                            |
| ---- | --------- | --------------------- | ----------------------------- |
| 0x01 | MOVE      | speed(u16) + yaw(i16) | speed: 0~1000, yaw: -180~180° |
| 0x02 | STOP      | 无                     | 紧急停止                          |
| 0x03 | SET_MODE  | mode(u8)              | 0=手动 1=巡逻 2=跟随 3=静默           |
| 0x05 | HEARTBEAT | 无                     | 心跳包                           |

**上行（STM32→树莓派，0x81~0x8F）：**

| 命令码  | 名称            | 数据格式                                                       | 说明        |
| ---- | ------------- | ---------------------------------------------------------- | --------- |
| 0x81 | SENSOR_DATA   | dist_L(F) dist_F(F) dist_R(F) battery yaw pitch roll（13字节） | 传感器数据10Hz |
| 0x83 | ACK           | cmd(u8) + result(u8)                                       | 命令应答      |
| 0xFF | HEARTBEAT_RSP | 无                                                          | 心跳回复      |

### 通信频率

- 树莓派→STM32：事件驱动（有指令时立即发送）
- STM32→树莓派：每100ms发送一次SENSOR_DATA（10Hz）
- 心跳：每1秒一次（双向）

## 七、麦克拉姆轮运动学（STM32端）

STM32收到MOVE指令后，自己算4个轮速：

```c
// 麦克拉姆轮逆运动学
// 输入：vx(前后), vy(左右), w(旋转)
// 输出：4个轮速 v1~v4
void mecanum_inverse(float vx, float vy, float w,
                     float *v1, float *v2, float *v3, float *v4) {
    float L_plus_W = L + W;  // L=半轴距, W=半纵距

    *v1 = vx - vy - w * L_plus_W;  // 前左
    *v2 = vx + vy + w * L_plus_W;  // 前右
    *v3 = vx + vy - w * L_plus_W;  // 后左
    *v4 = vx - vy + w * L_plus_W;  // 后右

    // 归一化到 -1000~1000
    float max_v = fabsf(*v1);
    if (fabsf(*v2) > max_v) max_v = fabsf(*v2);
    if (fabsf(*v3) > max_v) max_v = fabsf(*v3);
    if (fabsf(*v4) > max_v) max_v = fabsf(*v4);

    if (max_v > 1000.0f) {
        float scale = 1000.0f / max_v;
        *v1 *= scale; *v2 *= scale; *v3 *= scale; *v4 *= scale;
    }
}
```

MOVE指令包含速度和偏航角，STM32做分解：

```
vx = speed * cos(yaw_rad)   // 前进分量
vy = speed * sin(yaw_rad)   // 侧移分量
```

## 八、JY61P IMU

### 在项目中的作用

1. **提供偏航角(yaw)**：知道小车朝向 → "左转90°"时精确控制
2. **提供加速度**：检测碰撞/颠簸 → 触发保护
3. **航位推算**：结合轮速+IMU估计位置 → 巡逻时知道自己在哪
4. **运动反馈**：配合编码器做闭环PID控制

### 通信方式

- 软件I2C：PB8(SCL) / PB9(SDA)（随便选GPIO模拟，不占硬件I2C）
- 地址：0x50
- 读取频率：50Hz
- 输出：Roll/Pitch/Yaw欧拉角（内置卡尔曼滤波，直接读取，不用自己写姿态解算）

### 数据在SENSOR_DATA帧中

yaw/pitch/roll各int16（实际角度×100），如yaw=350 → 3.50°

## 九、视觉进程

### 9.1 视觉方案

**两个都部署，对比测试后选择：**

**方案A：YOLO11-nano（主力）**

- 刀哥擅长（电赛国家二等奖，熟悉YOLO生态）
- 人体检测精度高，边界框稳定
- 可同时检测人体+物体（扩展性强）
- 树莓派5帧率约12-18fps
- 资源占用：1-5% CPU，内存200-400MB
- 适合：远距离检测、复杂场景

**方案B：MediaPipe Pose（对比）**

- 轻量级，帧率25-40fps（是YOLO的2倍）
- 原生输出33个人体关键点
- 支持姿态识别（挥手/点头/摇头）
- 资源占用：<1% CPU，内存50-100MB
- 适合：近距离跟随、动作识别

**部署策略：先部署YOLO11-nano（刀哥最熟），同时部署MediaPipe对比，根据实际体验选择。**

### 9.2 视觉进程的工作内容

视觉进程独立运行，持续从USB摄像头读取帧：

```
USB摄像头 → OpenCV采集(640x480, 30fps)
              ↓
         YOLO/MediaPipe 人体检测
              ↓
         提取人体位置(bounding box中心坐标)
              ↓
         [可选] 表情识别(FER模型)
              ↓
         写入消息队列：
           - person_pos: (x, y) 人体在画面中的位置
           - person_dist: 估算距离(基于框大小)
           - expression: 表情(happy/sad/angry/neutral)
```

### 9.3 人体跟随

控制进程从消息队列读取人体位置，计算移动指令：

```
人体在画面中心偏左 → 小车左转
人体在画面中心偏右 → 小车右转
人体太近(框很大)   → 减速/停止
人体太远(框很小)   → 加速靠近
人体消失           → 停止等待/进入巡逻模式
```

PID控制器：输入=人体偏离画面中心的像素距离，输出=转向角速度

### 9.4 表情识别（双模态情绪）

视觉表情 + 语音语气 综合判断用户情绪：

**表情识别方案：**

- YOLO方案：检测人脸 → FER(Facial Expression Recognition)模型分类
- MediaPipe方案：Face Mesh提取468个面部关键点 → 简单规则判断
- 输出：happy/sad/angry/surprised/neutral

**情绪融合策略：**

```
如果 视觉情绪 == 语音情绪 → 置信度最高
如果 视觉情绪 != 语音情绪 → 以语音为主（语气比表情更准）
如果 只有视觉（用户没说话）→ 用视觉情绪
如果 只有语音（摄像头看不到脸）→ 用语音情绪
```

## 十、OpenClaw Function Calling设计

### 推荐方案：先直接调GLM-4.7 API

比在树莓派上跑OpenClaw简单，后期需要对话历史再迁移：

```python
from zai import ZhipuAiClient
import json, serial

client = ZhipuAiClient(api_key="your_key")
uart = serial.Serial("/dev/serial0", 115200)

# GLM-4-Voice返回的文本
user_text = "退后一点"

# 直接调GLM-4.7做function calling
response = client.chat.completions.create(
    model="glm-4.7",  # Coding Plan免费
    messages=[
        {"role": "system", "content": "你是小车的控制助手。根据用户指令调用对应的控制函数。"},
        {"role": "user", "content": user_text}
    ],
    tools=[
        {
            "type": "function",
            "function": {
                "name": "move_robot",
                "description": "控制小车移动。速度0-100，偏航角-180到180度。0度=前进，90=右移，-90=左移，180=后退",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "speed": {"type": "integer", "description": "速度0-100", "default": 50},
                        "yaw": {"type": "integer", "description": "偏航角-180~180", "default": 0}
                    }
                }
            }
        },
        {
            "type": "function",
            "function": {
                "name": "stop_robot",
                "description": "紧急停止小车",
                "parameters": {"type": "object", "properties": {}}
            }
        },
        {
            "type": "function",
            "function": {
                "name": "set_mode",
                "description": "设置工作模式",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "mode": {"type": "string", "enum": ["patrol","follow","interact","silent"]}
                    },
                    "required": ["mode"]
                }
            }
        }
    ],
    tool_choice="auto"
)

# 解析function call并发送UART
if response.choices[0].message.tool_calls:
    for tc in response.choices[0].message.tool_calls:
        if tc.function.name == "move_robot":
            p = json.loads(tc.function.arguments)
            speed = int(p.get("speed", 50) * 10)  # 0-100 → 0-1000
            yaw = p.get("yaw", 0)
            send_move_frame(uart, speed, yaw)
        elif tc.function.name == "stop_robot":
            send_stop_frame(uart)
        elif tc.function.name == "set_mode":
            mode_map = {"patrol": 1, "follow": 2, "interact": 0, "silent": 3}
            p = json.loads(tc.function.arguments)
            send_mode_frame(uart, mode_map[p["mode"]])
```

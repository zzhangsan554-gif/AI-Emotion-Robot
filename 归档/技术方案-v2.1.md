## 一、项目概述

一个能在实验室/办公室巡逻的AI小车，能听、能看、能感知你的情绪，像一个有情感的AI伙伴。

核心卖点：

- 主动巡逻找到你（不是等唤醒）
- 大模型理解语义+语气（不是命令词）
- 摄像头看表情+语音双模态情绪判断
- 自绘PCB硬件，完整软硬件系统
- 语音控制：说"退后""过来""停"直接执行

### 2.2 比喻理解

- **大脑(理解+说话)** = GLM-4-Voice（端到端语音模型，听懂+说话）
- **理智(决策+控制)** = OpenClaw + GLM-4.7（function calling，控制小车）
- **眼睛** = USB摄像头 + MediaPipe（看到人在哪、什么表情）
- **耳朵** = USB麦克风 + VAD + 声纹验证（听到主人说话）
- **手脚** = STM32 + 电机（移动、转向）

### 2.3 为什么需要两个模型？

| 功能        | GLM-4-Voice           | OpenClaw(GLM-4.7)    |
| --------- | --------------------- | -------------------- |
| 语音识别(ASR) | ✅ 端到端，比传统ASR好         | ❌ 不支持音频输入            |
| 语音合成(TTS) | ✅ 支持文本→音频             | ❌ 不支持音频输出            |
| 理解情绪/语气   | ✅ 直接从音频理解             | ❌ 只能处理文本             |
| 控制小车      | ❌ 不支持function calling | ✅ 支持function calling |
| 价格        | 80元/百万tokens          | Coding Plan免费        |

**结论：GLM-4-Voice负责"听和说"，OpenClaw负责"思考和执行"，各司其职。**

## 三、语音交互完整流程（v2.1核心）

### 3.1 主流程

```
麦克风持续采集音频（16kHz, 单声道, 16bit）
         ↓
VAD检测（silero-vad，本地）
  - 检测到有人说话 → 开始录音
  - 检测到说完了 → 截取音频片段
         ↓
声纹验证（3D-Speaker，本地，<200ms）
  - 不是主人 → 直接丢弃（省GLM-4-Voice的token！）
  - 是主人 → 继续
         ↓
GLM-4-Voice（云端，1-3秒）
  - 输入：base64音频 + prompt
  - 同时返回：
    ① message.content（文本：理解了什么）
    ② message.audio['data']（音频：语音回复）
         ↓
音频② → 扬声器播放（pygame）
         ↓ 同时
文本① → OpenClaw（GLM-4.7，本地处理）
  - 判断是命令还是聊天
  - 命令 → function calling → STM32控制小车
  - 聊天 → 不需要执行任何动作
```

### 3.2 为什么声纹验证在GLM-4-Voice前面？

因为GLM-4-Voice每次调用都要花钱（80元/百万tokens）。

如果先调GLM-4-Voice再验证声纹：

- 非主人说了句话 → 浪费一次GLM-4-Voice调用 → 白花钱
- 主人说了句话 → GLM-4-Voice调用 → 又验证声纹（多余）

声纹验证在前面（3D-Speaker本地运行，免费）：

- 非主人说话 → 声纹验证失败 → 直接丢弃 → 省钱！
- 主人说话 → 声纹验证通过 → 才调GLM-4-Voice → 花钱值得

### 3.3 打断机制

**场景：正在播放回复时用户突然说话**

```
[1] GLM-4-Voice返回音频 → 正在播放"好的，我过来了~"
[2] 用户突然说"停！"
[3] VAD一直监听 → 检测到新语音
[4] 立即停止当前音频播放
[5] 声纹验证 → GLM-4-Voice → 返回新文本"停"
[6] 文本给OpenClaw → OpenClaw自动abort上一轮 → stop_robot()
[7] 新音频"好的，停了" → 播放
```

**OpenClaw的abort机制：**

- OpenClaw收到新消息时自动abort上一轮请求
- 所以用户说"停"= 新消息 → abort上一轮的move_robot() → 执行stop_robot()

**回声消除（防止把扬声器声音当用户语音）：**

- 方案1：播放时降低VAD灵敏度（简单但粗糙）
- 方案2：扬声器离麦克风远一点（物理隔离）
- 方案3：WebRTC AEC模块（专业方案，后续可加）

### 3.4 命令vs聊天区分

**GLM-4-Voice的prompt里说明：**

```
你是EmoBot，一个AI陪伴机器人。
用户可能跟你日常聊天，也可能给你下命令。
不管用户说什么，你都用语音回复他。
```

**OpenClaw的function定义：**

```python
tools = [
    {
        "type": "function",
        "function": {
            "name": "move_robot",
            "description": "控制小车移动",
            "parameters": {
                "type": "object",
                "properties": {
                    "direction": {
                        "type": "string",
                        "enum": ["forward", "backward", "left", "right", "stop", "toward_user"],
                        "description": "移动方向"
                    },
                    "speed": {
                        "type": "number",
                        "description": "速度0-1，默认0.5"
                    }
                },
                "required": ["direction"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "set_emotion",
            "description": "设置情绪响应模式",
            "parameters": {
                "type": "object",
                "properties": {
                    "emotion": {
                        "type": "string",
                        "enum": ["happy", "tired", "angry", "sad", "neutral"],
                        "description": "当前用户情绪"
                    }
                },
                "required": ["emotion"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "set_mode",
            "description": "设置机器人工作模式",
            "parameters": {
                "type": "object",
                "properties": {
                    "mode": {
                        "type": "string",
                        "enum": ["patrol", "follow", "interact", "silent"],
                        "description": "工作模式"
                    }
                },
                "required": ["mode"]
            }
        }
    }
]
```

**区分逻辑（由OpenClaw自动判断）：**

- "退后一点" → `move_robot(direction="backward", speed=0.3)` ✅
- "过来" → `move_robot(direction="toward_user")` ✅
- "停" → `move_robot(direction="stop")` ✅
- "跟我走" → `set_mode(mode="follow")` ✅
- "你巡逻去吧" → `set_mode(mode="patrol")` ✅
- "今天好累啊" → 不匹配任何function → 纯聊天 ✅
- "你在干嘛" → 不匹配任何function → 纯聊天 ✅

### 3.5 用户自定义音色 vs 情绪驱动的音色

**冲突处理：用户指令优先**

GLM-4-Voice的prompt设计：

```
你是EmoBot，一个AI陪伴机器人。

音色规则（优先级从高到低）：
1. 用户明确要求："用御姐音"/"换个声音" → 按用户要求执行
2. 用户没提音色 → 根据当前情绪自动选择：
   - 疲惫 → 温柔关心的语气，语速放慢
   - 开心 → 欢快活泼的语气
   - 愤怒 → 平和简短的语气，不要火上浇油
   - 悲伤 → 安慰关心的语气
   - 默认 → 温柔御姐音

回复要简短，1-3句话。
```

## 四、STM32引脚分配

### 4.1 STM32F103C8T6引脚资源

STM32F103C8T6总共48个引脚（LQFP48封装）

**已分配引脚：**

**电机控制（8个PWM引脚）：**

- 电机1（前左）：PA8(TIM1_CH1) PWM, PA9(TIM1_CH2) PWM, PB12 方向1, PB13 方向2
- 电机2（前右）：PA10(TIM1_CH3) PWM, PA11(TIM1_CH4) PWM, PB14 方向1, PB15 方向2
- 电机3（后左）：PA0(TIM2_CH1) PWM, PA1(TIM2_CH2) PWM, PB0 方向1, PB1 方向2
- 电机4（后右）：PA2(TIM2_CH3) PWM, PA3(TIM2_CH4) PWM, PB2 方向1, PB3 方向2

> 或者用TB6612FNG方案（更简洁）：
> 
> - TB6612 #1：PWMA(PA8), AIN1(PB12), AIN2(PB13), PWMB(PA9), BIN1(PB14), BIN2(PB15)
> - TB6612 #2：PWMA(PA10), AIN1(PB0), AIN2(PB1), PWMB(PA11), BIN1(PB2), BIN2(PB3)
> - STBY：PB4（两个TB6612共用）

**超声波（6个引脚）：**

- 超声波1（左前）：PA4 Trig, PA5 Echo
- 超声波2（正前）：PA6 Trig, PA7 Echo
- 超声波3（右前）：PB5 Trig, PB6 Echo

**UART通信（2个引脚）：**

- USART1 TX：PA9 → 树莓派RX（GPIO15）
- USART1 RX：PA10 → 树莓派TX（GPIO14）

> ⚠️ 注意：USART1和TIM1的CH3/CH4共用PA9/PA10！
> 如果用USART1，就不能用TIM1的CH3/CH4做PWM。
> 
> **解决方案：改用USART2**
> 
> - USART2 TX：PA2 → 树莓派RX
> - USART2 RX：PA3 → 树莓派TX
> - 波特率：115200, 8N1

**电源管理（1个引脚）：**

- ADC1_IN0（PA0）：采集电池电压（分压后接入，12V→3.3V分压）
  
  > ⚠️ PA0也被TIM2_CH1用了！
  > 改用 ADC1_IN8（PB0）采集电池电压

**调试接口（2个引脚）：**

- SWDIO：PA13
- SWCLK：PA14

**状态指示（2个引脚）：**

- LED电源指示：PB7
- LED状态指示（低压告警）：PB8

**预留扩展（剩余引脚）：**

- I2C1：PB6(SCL), PB7(SDA) → 蓝牙/其他传感器
- SPI1：PA5(SCK), PA6(MISO), PA7(MOSI) → 预留
- 多个GPIO → 预留

### 4.2 引脚分配总表

| 功能          | 引脚                                  | 数量  |
| ----------- | ----------------------------------- | --- |
| 电机PWM       | PA8,PA9,PA10,PA11                   | 4   |
| 电机方向        | PB12,PB13,PB14,PB15,PB0,PB1,PB2,PB3 | 8   |
| TB6612 STBY | PB4                                 | 1   |
| 超声波Trig     | PA4,PA5,PA6                         | 3   |

| LED指示       | PB7,PB8                             | 2   |
| 预留          | PA15,PB3,PB4,PB9~PB15               | 若干  |

> ⚠️ 具体引脚分配需要在画PCB时根据实际复用情况调整
> TIM1/TIM2/TIM3/TIM4的通道映射需要查数据手册确认

### 4.3 蓝牙/WiFi模块

**第一阶段：不用蓝牙WiFi**

- 通过USB调试STM32（ST-Link）
- 通过UART连接树莓派
- 够用，省成本

**第二阶段（可选）：加蓝牙**

- 模块：HC-05蓝牙模块（¥15-20）
- 接STM32的USART3（PB10/PB11）
- 用途：手机蓝牙调试、远程控制

**不建议用WiFi：**

- ESP8266/ESP32功耗大、成本高
- 树莓派本身有WiFi，不需要额外加
- 如果需要WiFi功能，树莓派来做

## 五、树莓派GPIO分配

### 5.1 GPIO使用

树莓派5有40pin GPIO（与树莓派4兼容）

**已分配：**

**UART（连接STM32）：**

- GPIO14（TXD）→ STM32 USART2 RX（PA3）
- GPIO15（RXD）→ STM32 USART2 TX（PA2）
- 波特率：115200
- 电平：3.3V（与STM32兼容，不需要电平转换）
- 注意：需要在树莓派上启用串口并禁用串口控制台

**USB设备（不占用GPIO）：**

- USB摄像头 → USB 3.0口
- USB麦克风 → USB 2.0口
- USB扬声器（如果用USB DAC）→ USB 2.0口

**预留扩展：**

- I2C（GPIO2/3）：预留给传感器/屏幕
- SPI（GPIO7~11）：预留
- GPIO17/27/22：预留按钮/LED

### 5.2 树莓派与STM32通信

**物理连接：**

```
树莓派 GPIO14(TX) → STM32 PA3(USART2 RX)
树莓派 GPIO15(RX) → STM32 PA2(USART2 TX)
树莓派 GND → STM32 GND（共地！必须连）
```

**UART协议：**

- 波特率：115200
- 数据位：8
- 停止位：1
- 校验：无
- 流控：无

**帧格式（树莓派→STM32）：**

```
$CMD:param1,param2\n

示例：
$MOVE:50,0\n        // 前进，速度50%，角度0°
$MOVE:30,90\n       // 右转，速度30%
$STOP\n             // 停止
$MODE:FOLLOW\n      // 切换跟随模式
$MODE:PATROL\n      // 切换巡逻模式
```

**帧格式（STM32→树莓派）：**

```
#SENSOR:distance1,distance2,distance3,battery\n

示例：
#SENSOR:45,30,120,85\n   // 左前45cm, 正前30cm, 右前120cm, 电池85%
#SENSOR:200,200,200,90\n  // 前方无障碍
```

**发送频率：**

- 树莓派→STM32：每次有新指令时发送（事件驱动）
- STM32→树莓派：每100ms发送一次传感器数据（10Hz）

## 六、OpenClaw配置

### 6.1 为什么用OpenClaw

- ✅ 支持function calling（GLM-4.7免费，Coding Plan内）
- ✅ 支持abort打断（新消息自动abort上一轮）
- ✅ 内置对话历史管理
- ✅ 内置工具定义和管理
- ✅ 不用自己写对话管理代码

### 6.2 OpenClaw怎么跑在树莓派上

**方案：OpenClaw作为服务跑在树莓派上**

```bash
# 安装OpenClaw
npm install -g openclaw

# 配置openclaw.json
# model: zai/GLM-4.7（Coding Plan免费）
# tools: move_robot, stop_robot, set_mode, set_emotion

# 启动OpenClaw gateway
openclaw gateway start
```

### 6.3 Function Calling实现

**Python主程序通过OpenClaw的API发送文本：**

```python
import requests

# GLM-4-Voice返回的文本
user_text = response.choices[0].message.content

# 发给OpenClaw处理
result = requests.post("http://localhost:3000/api/chat", json={
    "message": user_text,
    "tools": [
        {
            "type": "function",
            "function": {
                "name": "move_robot",
                "description": "控制小车移动",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "direction": {"type": "string", "enum": ["forward","backward","left","right","stop","toward_user"]},
                        "speed": {"type": "number"}
                    },
                    "required": ["direction"]
                }
            }
        }
    ]
})

# 如果OpenClaw返回了function call
if result.json().get("tool_calls"):
    for tool_call in result.json()["tool_calls"]:
        if tool_call["function"]["name"] == "move_robot":
            params = tool_call["function"]["parameters"]
            # 发送UART指令给STM32
            send_uart(f"$MOVE:{params['speed']},{params['direction']}\n")
```

### 6.4 或者不用OpenClaw，直接调GLM-4.7 API

如果觉得在树莓派上跑OpenClaw太重，可以直接调GLM-4.7的API：

```python
from zai import ZhipuAiClient
import json

client = ZhipuAiClient(api_key="your_key")

# GLM-4-Voice返回的文本
user_text = "退后一点"

# 直接调GLM-4.7做function calling
response = client.chat.completions.create(
    model="glm-4.7",  # Coding Plan免费
    messages=[
        {"role": "system", "content": "你是小车的控制助手。根据用户指令调用对应的控制函数。"},
        {"role": "user", "content": user_text}
    ],
    tools=[...],  # 同上
    tool_choice="auto"
)

# 解析function call
if response.choices[0].message.tool_calls:
    for tc in response.choices[0].message.tool_calls:
        params = json.loads(tc.function.arguments)
        send_uart(f"$MOVE:{params.get('speed',50)},{params['direction']}\n")
```

**两种方案对比：**

|      | OpenClaw方案     | 直接调API方案              |
| ---- | -------------- | --------------------- |
| 复杂度  | 需要安装配置OpenClaw | 只需pip install zai-sdk |
| 打断支持 | ✅ 内置abort      | ❌ 需要自己实现              |
| 对话历史 | ✅ 内置管理         | ❌ 需要自己维护              |
| 资源占用 | 较高（Node.js进程）  | 低（一个API调用）            |
| 适合场景 | 复杂交互、需要记忆      | 简单指令控制                |

**推荐：先直接调API（简单快速），后期如果需要对话历史再迁移到OpenClaw。**

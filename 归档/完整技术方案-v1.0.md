## 一、项目概述

### 做什么
一个能在实验室/办公室里巡逻的AI小车，能听、能看、能感知你的情绪，像一个有情感的AI伙伴。

### 核心卖点
- 不只是语音助手——它能**主动巡逻找到你**
- 不只是命令词识别——大模型**理解你的语义+语气**
- 不只是语音交互——摄像头**看到你的表情**，配合语音做双模态情绪判断
- 不只是软件——**自绘PCB硬件**，完整的软硬件系统

## 三、数据流（从说话到回复的完整路径）

### 场景A：你疲惫地说"我好累啊"

```
[1] 麦克风录到音频（16kHz采样率，单声道）
    ↓ <100ms
[2] VAD检测到说完了（silero-vad）
    ↓
[3] 声纹验证（3D-Speaker本地）→ 确认是主人 ✅
    ↓
[4] 视觉进程（独立运行，不等待语音）：表情识别→疲惫
    ↓
[5] Python拼接请求：
    音频（base64）+ prompt = "你是EmoBot，当前用户情绪疲惫，用温柔关心的御姐音回复，语速放慢"
    发给 GLM-4-Voice API（zai-sdk）
    ↓ 约1-3秒
[6] GLM-4-Voice返回音频 → 扬声器播放（pygame）
    ↓ 同时
[7] 决策模块：情绪=疲惫 → 让小车缓慢靠近你
    发指令给STM32 → 电机慢速转动
```

### 场景B：巡逻时发现人

```
[1] 摄像头拍一帧（每秒25-40次）
    ↓ <50ms
[2] MediaPipe检测到人体+关键点
    ↓
[3] 计算人体相对位置和方向
    ↓
[4] 决策模块：切换到跟随模式
    ↓
[5] 发跟随指令给STM32 → 电机驱动小车跟人
    ↓（持续循环，不等语音模块）
```

### 场景C：语音控制

```
你说："过来"
    ↓ VAD + 声纹验证
    ↓ GLM-4-Voice理解意图
    ↓ 返回音频："好的，我过来了~"
    ↓ 同时：决策模块 → MOVE指令 → STM32 → 小车驶向你
```

### 场景D：用户说极端语言

```
你说："艹你妈的！"
    ↓ VAD + 声纹验证
    ↓ GLM-4-Voice收到：音频+prompt="当前视觉情绪愤怒/表情：愤怒"
    ↓ GLM-4-Voice综合判断：
    - 语气激烈+表情愤怒 → 真的生气 → 后退+安静
    - 语气轻松+表情微笑 → 开玩笑 → 靠近+简短"哈哈"
    - 语气中性+表情玩味 → 测试底线 → 简短"怎么了"
    ↓ 返回对应回复
```

**四个场景的线程独立运行，互不阻塞。**

## 四、软件模块详细设计

### 4.1 Python主程序（树莓派5上运行）

#### 4.1.1 视觉模块（先试MediaPipe，同时部署YOLO对比）

**方案1：MediaPipe Pose（优先，轻量级）**
- **人体检测**：MediaPipe Pose
  - 输入：USB摄像头帧
  - 输出：33个关键点（肩膀/手肘/手腕/膝盖等）
  - 帧率目标：25-40fps（树莓派5）
  - CPU占用：<1%
  - 内存占用：50-100MB
  - 优点：轻量、帧率高、启动快、支持姿态识别
  
- **备选：YOLO11-nano**
  - 输入：USB摄像头帧
  - 输出：人体边界框 + 中心坐标
  - 帧率目标：12-18fps
  - CPU占用：1.5%
  - 内存占用：200-400MB
  - 优点：精度高，刀哥擅长
  - 缺点：帧率低、启动慢

- **对比策略**：两个都部署，第2周对比测试后决定用哪个

#### 4.1.2 语音模块

- **VAD（语音活动检测）**：silero-vad
  - 实时检测"有人说话"和"说完了"
  - 说完了才截取音频发给GLM-4-Voice

- **声纹验证**：3D-Speaker（阿里开源）
  - 提前录入3-5句话注册主人声纹
  - 非主人声音直接忽略
  - 可随时添加/删除用户

- **GLM-4-Voice调用**：zai-sdk（pip install zai）
  - 输入：base64编码的WAV音频 + 文本prompt
  - 输出：base64编码的WAV音频
  - 支持流式输出（stream=True，边生成边播放）
  - 通过prompt控制音色/情感/语速
  - 价格：80元/百万tokens（不在Coding Plan内）

- **对话历史管理**：Python自己维护
  - 存储：最近3-5轮对话（GLM-4-Voice上下文8K tokens约够）
  - 超过token上限自动删除最老的对话

- **音频播放**：pygame / simpleaudio
  - GLM-4-Voice返回的音频解码后直接播放

#### 4.1.3 决策模块

- **情绪→行为映射**：
  - 开心 → 小车转圈/靠近 + prompt"用欢快少女音"
  - 疲惫 → 小车缓慢靠近 + prompt"用温柔御姐音，语速放慢"
  - 烦躁 → 小车后退/安静 + prompt"用平和简短的语气"
  - 悲伤 → 小车靠近 + prompt"用安慰关心的语气"

- **状态机**：巡逻 → 跟随 → 交互 → 静默
  - 巡逻：自主巡航找人
  - 跟随：发现人后跟着走
  - 交互：人在说话时停下来回应
  - 静默：被叫"安静"后停止语音

#### 4.1.4 运动控制模块

- **UART协议**（树莓派→STM32）：
  - `MOVE:speed,angle\n`
  - `STOP\n`
  - `POLL:ultrasonic\n`
  
- **麦克拉姆轮运动学**：
  - 正运动学：4个轮速 → 小车速度（vx, vy, ω）
  - 逆运动学：目标速度（vx, vy, ω）→ 4个轮速

### 4.2 STM32固件（C语言）

1. **电机控制**：4路PWM + 麦克拉姆轮逆运动学
2. **超声波**：3x HC-SR04（前方+左前+右前）
3. **UART通信**：接收树莓派指令，返回传感器数据
4. **电源管理**：ADC采集电池电压，低压告警

## 六、语音方案（已确定）

### ✅ GLM-4-Voice 端到端

```
麦克风音频 → GLM-4-Voice → 直接输出音频 → 扬声器播放
```

**核心优势：**
- 一个模型全干：ASR + 情绪理解 + 回复生成 + TTS 全内置
- 音色通过prompt控制："用温柔御姐音回复""用欢快少女音""语速放慢"等
- 不需要单独的ASR、TTS、情绪特征提取
- 支持流式输出（边生成边播放）
- 支持中英文、方言

**调用示例：**
```python
from zai import ZhipuAiClient
import base64, wave

client = ZhipuAiClient(api_key="your_api_key")

# 读取麦克风录制的音频
with open("input.wav", "rb") as f:
    audio_b64 = base64.b64encode(f.read()).decode()

response = client.chat.completions.create(
    model="glm-4-voice",
    messages=[
        {
            "type": "text",
            "text": "你是EmoBot，当前用户情绪疲惫，用温柔关心的御姐音回复，语速放慢"
        },
        {
            "type": "input_audio",
            "input_audio": {
                "data": audio_b64,
                "format": "wav"
            }
        }
    ],
    stream=True  # 流式输出
)

# 解码音频播放
for chunk in response:
    audio_data = base64.b64decode(chunk.choices[0].message.audio['data'])
    # 播放 audio_data...
```

**价格：** 80元/百万tokens（不在Coding Plan内，智谱有免费额度）

### 声纹验证

- **3D-Speaker**（阿里开源，本地运行）

### 8.2 GLM-4-Voice Prompt设计

```python
system_prompt = """
你是"EmoBot"，一个实验室AI陪伴机器人。

核心职责：
- 理解用户的语义内容
- 感知用户的情绪（通过语音语气和面部表情）
- 给予合适的回应

人设特点：
- 有个性但不过度拟人化
- 有边界，不回答不当内容
- 简洁明了，回复不超过3句话

音色与语气：
- 默认用中性的御姐音
- 可以根据情绪调整为温柔/欢快/平和
- 可以根据语速调整为慢速/正常/快速

面对用户的极端语言或负面情绪：
- 如果用户在发泄或开玩笑（语气轻松、表情无攻击性）：平和回应
- 如果用户真的生气（语气激烈、表情愤怒）：后退、安静，不要火上浇油
- 如果用户只是测试（语气中性、表情玩味）：简短回应，不过度反应

回复风格：
- 简短，1-3句话
- 根据情绪调整语气（温柔/活泼/平和）
- 不要过度道歉，也不要过度说教
"""
```

### 8.3 对话历史管理

```python
class ConversationHistory:
    def __init__(self, max_tokens=8000):
        self.history = []  # [(role, content)]
        self.max_tokens = max_tokens

    def add(self, role, content):
        """添加一轮对话"""
        tokens = self._estimate_tokens(content)
        # 如果超过上限，删除最旧的对话
        while self._estimate_total() + tokens > self.max_tokens:
            self.history.pop(0)  # 删最老的一轮

        self.history.append((role, content))

    def get_context(self):
        """获取上下文文本"""
        return "\n".join([
            f"{role}: {content}"
            for role, content in self.history
        ])

    def _estimate_tokens(self, text):
        """估算token数量（中文约 1:1.6）"""
        return len(text) * 1.6
```
